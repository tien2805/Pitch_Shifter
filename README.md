# FPGA Real-time Pitch Shifter (CORDIC DSP Engine)

A real-time audio pitch shifting system implemented on FPGA, utilizing a 16-stage CORDIC pipeline architecture. The design performs Double Sideband Suppressed-Carrier (DSB-SC) modulation without relying on dedicated hardware multipliers (Zero DSP48 Multipliers).

## Overview

This project designs a Digital Signal Processing (DSP) core running entirely on FPGA hardware, capable of real-time audio frequency shifting with ultra-low latency (~42 µs).

**Operating Principle:** 24-bit audio from an I2S Microphone is received -> DC Offset Filtering -> Multiplied by a Cosine carrier generated by CORDIC -> Output to DAC/Speaker via I2S. The multiplication `Audio × cos(θ)` creates a frequency shifting effect (DSB-SC modulation).

## System Architecture

```text
┌─────────┐    ┌──────────┐    ┌────────────────────────────────────────┐    ┌──────────┐    ┌─────────┐
│  MIC    │───▶│  I2S RX  │───▶│           DSP CORE (sys_clk)           │───▶│  I2S TX  │───▶│  DAC/   │
│ (I2S)   │    │ (bclk)   │    │                                        │    │ (bclk)   │    │  Speaker│
└─────────┘    └────┬─────┘    │  ┌───────────┐  ┌──────────────────┐  │    └──────────┘    └─────────┘
                    │          │  │DC Remover  │  │  CORDIC Core     │  │
               ┌────▼─────┐   │  │(IIR HPF)   │─▶│  (16-stage pipe) │  │
               │Async FIFO│──▶│  └───────────┘  └──────────────────┘  │───▶┌──────────┐
               │(Gray CDC)│   │  ┌───────────────────┐                │    │Async FIFO│
               └──────────┘   │  │Phase Accumulator   │                │    │(Gray CDC)│
                              │  │(DDS Oscillator)    │                │    └──────────┘
                              │  └───────────────────┘                │
                              └────────────────────────────────────────┘
```

### Dual Clock Domain Management

| Clock Domain | Frequency | Function |
|---|---|---|
| `bclk` (I2S) | 12.288 MHz | Serial communication with Mic/DAC |
| `sys_clk` (System) | 50 MHz | CORDIC DSP Core processing |

The two clock domains are isolated by an **Async FIFO** using Gray code pointers and 2-FF Synchronizers to prevent metastability.

## Key Features

| Feature | Description |
|---|---|
| **Zero DSP48** | Sine/Cosine calculations use shift-and-add (CORDIC), avoiding hardware multipliers. |
| **Ultra-low Latency** | 16-stage pipeline + 2 I/O stages = ~18 clock cycles ≈ **42 µs**. |
| **Quadrant Mapping** | Full 360° phase mapping before CORDIC processing. |
| **Gain Compensation** | K ≈ 0.60742 compensated via bit-shifting: `(x>>>1)+(x>>>3)-(x>>>6)-(x>>>9)`. |
| **Dual Clock Domain** | Async FIFO with Gray Code CDC between I2S and System clocks. |
| **Stereo Processing** | Parallel processing for Left/Right channels using two independent CORDIC instances. |

## Simulation Guide

### Step 1: Generate Input Data
```bash
cd sim
python plot_demo.py generate
```
Generates `demo_input.hex` containing a dual sine wave (500 Hz + 1200 Hz).

### Step 2: Run ModelSim Simulation
```tcl
# Inside ModelSim Transcript window:
cd <path_to_project>/sim
do demo_run.do
```
This script automatically compiles the RTL, loads the testbench, and displays the analog waveforms.

### Step 3: Plot Analysis
```bash
python plot_demo.py plot
```
Outputs Time Domain (Input), Time Domain (Output with Envelope), and FFT Spectrum graphs.
